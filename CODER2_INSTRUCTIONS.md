# 코더2 (Gemini 3.0 Pro) 작업 지시서

## 역할
프론트엔드 코딩 담당

## 첫 번째 과업: 기존 코드 확인 및 검토

다음 파일들을 확인하고 검토하세요:

1. **lunch-service/index.html** (레포지토리 2)
   - HTML 구조 확인
   - 모바일 최적화 설정 (meta viewport) 확인
   - 추천/목록/등록 탭 구조 확인
   - 하단 고정 탭바 구조 확인
   - 필요한 요소들이 모두 포함되어 있는지 확인

2. **lunch-service/lunch.css** (레포지토리 2)
   - 스타일 구조 확인
   - 모바일 최적화 (max-width: 420px, 중앙 정렬) 확인
   - body 배경 회색 (#f5f5f5) 설정 확인
   - 하단 탭바 고정 스타일 (position: fixed) 확인
   - 카드 스타일 확인
   - 반응형 디자인 확인

3. **lunch-service/lunch.js** (레포지토리 2)
   - 파일 존재 여부 확인
   - 존재한다면 코드 확인
   - 존재하지 않는다면 새로 작성 필요

## 작업 수행 방법

1. 위 파일들을 하나씩 확인하며 검토
2. 완료된 부분과 미완성 부분을 구분
3. 미완성 부분이나 개선이 필요한 부분을 TASKS.md의 체크리스트에 맞춰 작업
4. 작업 완료 시 WORKLOG.md에 기록

## 주요 구현 사항

### lunch.js에 포함해야 할 기능:
- API 호출 함수 (`API_BASE_URL` 환경변수 사용)
- 추천 탭 로직 (자연어 입력, 프리셋 선택, 결과 표시)
- 목록 탭 로직 (검색/필터, 카드 리스트)
- 등록 탭 로직 (폼 제출, 유효성 검사)
- 탭 전환 로직
- 로딩 상태 관리
- 에러 처리 및 토스트 메시지

## 참고 문서

- `API_SPEC.md`: API 인터페이스 정의
- `TASKS.md`: 전체 작업 할당 및 체크리스트
- `WORKLOG.md`: 작업 진행 상황 로그
