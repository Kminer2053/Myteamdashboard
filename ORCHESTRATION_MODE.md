# 오케스트레이션 모드 설정

## 기본 원칙

**하이브리드 방식 (3번 방식)**을 사용합니다.

- 오케스트레이터: 계획 수립, 작업 배분, 문서 작성, 코드 리뷰
- 코더1 (백엔드): 별도 채팅에서 백엔드 작업 수행
- 코더2 (프론트엔드): 별도 채팅에서 프론트엔드 작업 수행

## 협업 방식

1. 오케스트레이터가 TASKS.md에 작업을 배분
2. 각 코더가 별도 채팅에서 자신의 작업 수행
3. WORKLOG.md에 진행 상황 기록
4. 오케스트레이터가 코드 리뷰 및 통합

## 공유 문서

- `WORKLOG.md`: 작업 진행 상황 로그
- `TASKS.md`: 작업 할당 및 체크리스트
- `API_SPEC.md`: API 인터페이스 정의
